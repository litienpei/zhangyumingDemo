一. mysql的比较详解
    1.1 mysql比较英文字符串
        -命令:
            SELECT 'abc'<'abd' 输出: 1
        -说明:
            mysql比较字符串的原理和各个编程语言一样, 按照ASCII码值对应表按位比较每一位, 哪一位大了立即终止比较.
    1.2 mysql比较中文字符串
        -命令:
            SELECT '账'>'张'  输出: 1
            SELECT '章'>'张'  输出: 1
            SELECT '章'>'账'  输出: 0
        -说明:
            --编码简介
                根据输出结果, 可以得知: 账 > 章 > 张, 但是mysql是怎么比较中文的呢? 首先想到的应该会是拼音, 但是这三个字是同
                音的, 并且外国人是不会拼音的, 这里就涉及到字符编码相关的知识, 不同的编码对中文的处理不一样,例如:
                    --'张'在Unicode和utf-8下的编码值:
                        Unicode : \u5f20
                        utf-8 : &#x5F20;
                    --说明:
                        不同编码对于汉字的处理不同, 中文在计算机中并不是以中文形式保存, 而是翻译为对应的编码进行保存, 所以,
                        我的电脑的mysql的编码设置为了uft-8, 所以我得电脑在比较字符串的时候按照utf-8编码进行比较的.
            --张, 章, 账三个字的uft-8编码
                张 : &#x5F20;
                章 : &#x7AE0;
                账 : &#x8D26;
                ---说明
                    可以发现, utf-8编码下的这三个字的编码大小确实是: 账 > 章 > 张.
